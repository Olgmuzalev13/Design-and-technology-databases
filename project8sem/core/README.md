# Core приложение

Данная директория содержит основное Django-приложение `core`, реализующее базовую функциональность учебной АСУ ТП: работу с пользователями, сотрудниками и пользовательскими интерфейсами. Приложение построено по стандартной архитектуре Django и предназначено для изучения принципов построения web-интерфейсов и серверной логики в рамках курса 8 семестра МГТУ им. Баумана.

## Общая роль приложения `core`

Приложение `core` выполняет роль базового модуля системы:

* хранение доменных моделей (сотрудники, расширение пользователя),
* обработка HTTP-запросов,
* отображение HTML-шаблонов,
* реализация простейшей аутентификации и регистрации.

Данная структура может быть расширена и использована как основа для любой АСУ ТП.

---

## Структура каталога

```
core/
├── migrations/
├── templates/
│   └── core/
├── apps.py
├── models.py
├── tests.py
├── views.py
└── README.md
```

Ниже приведено подробное описание назначения каждого элемента.

---

## models.py

Файл `models.py` содержит **модели данных**, которые описывают структуру таблиц базы данных.

В терминах АСУ ТП:

* модель = сущность предметной области,
* поле модели = атрибут сущности.

Примеры:

* сотрудник,
* должность,
* подразделение.

Модели Django автоматически преобразуются в таблицы PostgreSQL с помощью механизма миграций. Любое изменение моделей требует создания и применения миграций.

Важно:

* модели не должны содержать логику отображения,
* бизнес-логика здесь должна быть минимальной и обоснованной.

---

## views.py

Файл `views.py` содержит **представления**, которые обрабатывают входящие HTTP-запросы.

Задачи представлений:

* принять запрос от браузера,
* при необходимости получить данные из базы,
* передать данные в шаблон,
* вернуть HTTP-ответ.

В учебном проекте используются в основном **function-based views**, так как они проще для понимания на начальном этапе.

Важно:

* представления не должны содержать HTML-код,
* вся визуальная часть выносится в шаблоны.

---

## templates/core/

Каталог `templates/core/` содержит **HTML-шаблоны**, отвечающие за пользовательский интерфейс системы.

Шаблоны используют Django Template Language и позволяют:

* переиспользовать общий макет страницы (base.html),
* вставлять данные, переданные из `views.py`,
* избегать дублирования HTML-кода.

Типичные шаблоны:

* `base.html` - базовый шаблон с навигацией и футером,
* `login.html` - форма входа,
* `register.html` - форма регистрации,
* страницы со списками данных.

Важно:

* шаблоны не должны содержать Python-код,
* вся логика принятия решений должна быть минимальной.

---

## admin.py

Файл `admin.py` отвечает за регистрацию моделей в **Django Admin Panel**.

Django Admin используется:

* для отладки,
* для просмотра и редактирования данных,
* как инструмент быстрого администрирования.

Регистрация моделей здесь не влияет на работу пользовательского интерфейса.

---

## apps.py

Файл `apps.py` содержит конфигурацию приложения `core`.

Он используется Django:

* при инициализации приложения,
* для подключения сигналов и расширений.

Как правило, данный файл изменяется редко.

---

## migrations/

Каталог `migrations/` содержит файлы миграций базы данных.

Миграции:

* описывают изменения структуры БД,
* позволяют синхронизировать код и базу данных,
* обязательны для применения после изменения моделей.

Важно:

* миграции являются частью проекта и должны коммититься в репозиторий,
* ручное редактирование миграций допускается только при понимании последствий.

---

## tests.py

Файл `tests.py` предназначен для размещения автоматических тестов.

В рамках учебного проекта:

* тесты используются для проверки корректности работы логики,
* покрытие тестами контролируется CI.

Наличие тестов является обязательным требованием для допуска изменений в основную ветку проекта.

---

## Общие замечания

* Приложение построено с учетом дальнейшего масштабирования.
* Структура соответствует рекомендациям Django и промышленным практикам.
